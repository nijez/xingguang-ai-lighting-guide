# 馨光 AI 设计灯光分层验收标准

更新时间：2026-06-29

只有 L1-L6 全部通过，才算“馨光 AI 设计灯光”主链路成功。脚本显示 100%、接口返回 200、Skill ready 都只是局部条件，不能单独代表项目成功。

## L1 基础环境

通过标准：

- OpenClaw CLI 存在。
- OpenClaw CLI 和 gateway 使用兼容版本。
- gateway service active running。
- connectivity probe ok。
- lightclawbot 连接正常。

失败或警告：

- CLI 与 gateway 版本不一致。
- gateway service 仍指向旧版 ExecStart。
- gateway 未运行。
- lightclawbot 未连接。

## L2 米家绑定

通过标准：

- 米家账号已绑定。
- 家庭列表可读取。
- 多家庭时用户已明确选择目标家庭。
- `target-home.env` 正确保存目标家庭。

失败或警告：

- 未绑定米家。
- 家庭列表读取失败。
- 多家庭时自动使用第一个家庭。
- 目标家庭缺失。

## L3 灯光服务

通过标准：

- `xinguang-wainfort.service` active running。
- `/api/status` 返回 200。
- `/api/devices` 不返回 500。
- Miloco 授权信息已同步到灯光服务。
- root fallback 只用于 wainfort-server 服务，不扩大到整个安装流程。

失败或警告：

- 灯光服务未运行。
- `/api/status` 不可访问。
- `/api/devices` 返回 401，通常代表授权未同步或过期。
- `/api/devices` 返回 500。
- env 文件权限不是 600。

## L4 设备发现

通过标准：

- 能读取当前家庭设备列表。
- 能看到目标家庭里的馨光设备。

警告：

- `/api/devices` 返回 200 但 `devices=""`。
- 设备列表为空但接口本身可访问。

失败：

- 设备接口不可访问。
- 设备接口返回 401 或 500。

## L5 本地控灯

通过标准：

- `xinguang-test-scene "客厅" "马尔代夫海边日落"` 能发出单次灯光请求。
- 本地服务收到请求。
- 调用链路可观测。
- 用户现场确认灯光变化。

失败或警告：

- 本地服务未收到请求。
- 请求未发出。
- 请求发出但现场无变化。
- 返回值异常但现场有变化时，应记录为物理成功、返回值需研发修复。

## L6 龙虾自然语言控灯

通过标准：

- 用户说：`客厅来个马尔代夫的海边日落`。
- 龙虾调用馨光 Skill。
- 本地灯光服务收到请求。
- 馨光设备实际变化。
- 龙虾不输出误导性错误，也不虚假承诺“已成功”。

失败或警告：

- 龙虾未调用 Skill。
- 龙虾进入 ClawHub 搜索。
- 龙虾推荐普通米家灯控语句。
- 龙虾泄露 token、路径、端口、PID、日志、内部命令。
- 灯光无变化。

## 结论规则

- L1 FAIL：先修基础环境。
- L2 FAIL：先完成米家绑定和目标家庭选择。
- L3 FAIL：先修灯光服务和授权同步。
- L4 WARN：可以继续排查，但不能宣称设备发现成功。
- L5 未验收：不能宣称本地控灯链路成功。
- L6 未验收：不能宣称用户自然语言主链路成功。
