第二阶段真实服务器测试指令

测试前提：
1. 第一阶段已完成。
2. 小龙虾可以正常对话。
3. MiMo API Key 已配置。
4. 米家账号已绑定。
5. 目标家庭为：林坞店。
6. 单次灯光测试目标为：二楼客厅 / RGBCW灯柱。

说明：上面的房间和设备只用于真实服务器单次测试，不是正式用户流程。正式用户流程只需要选择米家家庭，之后直接用自然语言描述灯光需求，由馨光 Skill 处理设备匹配和灯光执行。

本测试拆成两段执行。

第一段：安装和识别

目标：
1. 下载第二阶段脚本。
2. 校验 `XINGUANG_SKILL_INSTALLER_VERSION="2026-06-26.3"`。
3. 自动下载 Skill 文件。
4. 自动安装 Skill。
5. 自动下载并启动 `wainfort-server`。
6. 自动读取所选家庭下的设备列表。
7. 输出安装结果。
8. 只确认是否可以进入自然语言灯光控制，不要自动控制灯光。

请把下面内容发给小龙虾执行：

```text
请你执行「馨光 Skill 安装与家庭设备列表准备」。全程必须使用中文回复。

重要要求：
1. 不要重复执行第一阶段安装流程。
2. 不要修改第一阶段安装脚本。
3. 不要自行搜索网络，不要切换其它安装方式。
4. 本次目标是安装馨光 Skill，让小龙虾可以通过馨光 Skill 控制馨光设备。
5. 不要输出或保存用户密码、授权信息、MiMo API Key。
6. 安装任务启动后，不要长时间占用对话。
7. 不要重复发送本安装指令。
8. 如果检测到多个米家家庭，请让我选择要控制馨光设备的家庭，不要自动使用第一个家庭。
9. 本段只选择家庭并读取该家庭下的设备列表，不要让我做额外的灯光设备选择。
10. 本段目标家庭为「林坞店」。
11. 本段只做安装和设备列表读取，不要控制灯光。

请只执行下面这份固定流程：

TARGET=/tmp/install-xinguang-skill.sh
LOG=/tmp/xinguang-skill-install-current.log
STATE=/tmp/xinguang-skill-install.state
PIDFILE=/tmp/xinguang-skill-install.pid
EXPECTED_VERSION=2026-06-26.3
REJECT_VERSION_1=2026-06-26.1
REJECT_VERSION_2=2026-06-26.2

if [ -f "$PIDFILE" ]; then
  OLD_PID="$(cat "$PIDFILE" 2>/dev/null || true)"
  if [ -n "$OLD_PID" ] && ps -p "$OLD_PID" >/dev/null 2>&1; then
    echo "馨光 Skill 安装任务已经在运行，请不要重复启动。"
    echo "如需查看进度，请发送：查看馨光 Skill 安装进度"
    exit 0
  fi
fi

rm -f "$TARGET"
for URL in \
  https://nijez.github.io/xingguang-ai-lighting-guide/install-xinguang-skill.sh \
  https://raw.githubusercontent.com/nijez/xingguang-ai-lighting-guide/main/install-xinguang-skill.sh \
  https://cdn.jsdelivr.net/gh/nijez/xingguang-ai-lighting-guide@main/install-xinguang-skill.sh
do
  echo "尝试下载馨光 Skill 安装脚本: $URL"
  if curl -fsSL "$URL" -o "$TARGET" && grep -q "XINGUANG_SKILL_INSTALLER_VERSION=\"$EXPECTED_VERSION\"" "$TARGET"; then
    echo "馨光 Skill 安装脚本下载成功，版本 $EXPECTED_VERSION"
    break
  fi
  echo "当前源失败或版本不匹配，继续尝试下一个源"
done

grep -q "XINGUANG_SKILL_INSTALLER_VERSION=\"$REJECT_VERSION_1\"" "$TARGET" 2>/dev/null && { echo "拒绝使用旧版 $REJECT_VERSION_1，当前必须使用 $EXPECTED_VERSION"; exit 1; }
grep -q "XINGUANG_SKILL_INSTALLER_VERSION=\"$REJECT_VERSION_2\"" "$TARGET" 2>/dev/null && { echo "拒绝使用旧版 $REJECT_VERSION_2，当前必须使用 $EXPECTED_VERSION"; exit 1; }
grep -q "XINGUANG_SKILL_INSTALLER_VERSION=\"$EXPECTED_VERSION\"" "$TARGET" || { echo "馨光 Skill 安装脚本下载失败或版本不正确，当前必须使用 $EXPECTED_VERSION"; exit 1; }
chmod +x "$TARGET"

nohup env \
  XINGUANG_TARGET_HOME="林坞店" \
  INSTALL_ACTION=continue \
  INSTALL_NONINTERACTIVE=1 \
  bash "$TARGET" >> "$LOG" 2>&1 &
echo $! > "$PIDFILE"
sleep 2

echo "馨光 Skill 后台安装已启动"
echo "安装脚本版本: $EXPECTED_VERSION"
echo "日志文件: $LOG"
echo "状态文件: $STATE"
echo "说明: 安装任务已在服务器后台运行。请稍后发送「查看馨光 Skill 安装进度」。不要重复发送本安装指令。"
```

第二段：单次灯光测试

只有在第一段确认「馨光 Skill 已安装、灯控服务已运行、目标家庭下设备列表已读取」后，才发送第二段。

测试目标：

```text
家庭：林坞店
房间：二楼客厅
设备：RGBCW灯柱
测试效果：森林晨光
```

请把下面内容发给小龙虾执行：

```text
请执行「馨光 Skill 单次灯光测试」。全程必须使用中文回复。

重要要求：
1. 只控制「林坞店 / 二楼客厅 / RGBCW灯柱」这个指定设备。
2. 只发送一次「森林晨光」灯光请求。
3. 请求发送后立即停止，等待我观察现场灯光。
4. 不要自动重试。
5. 不要自动关灯。
6. 不要自动开灯。
7. 不要切换其它颜色继续验证。
8. 不要控制其它设备做对照。
9. 不要调用 `miloco-cli device control` 直接控制 RGBCW 设备。
10. 不要仅根据 API `success` 字段判断成败。

可以做的只读检查：
1. 服务是否运行。
2. Skill 是否安装。
3. 设备列表里是否有目标设备。

请按下面流程执行：
1. 确认目标设备是「林坞店 / 二楼客厅 / RGBCW灯柱」。
2. 记录 `LIGHT_TEST_SINGLE_SHOT`。
3. 只通过馨光 Skill 发送一次「森林晨光」灯光请求。
4. 请求发送后记录 `LIGHT_REQUEST_SENT`。
5. 如果 API 返回 `success:false`，记录 `LIGHT_API_RETURNED_FALSE`，但不要继续控制设备。
6. 回复：

灯光请求已发送，请观察目标设备是否发生变化。
如果灯光已变化，请回复“已变化”。
如果灯光未变化，请回复“未变化”。
```

如果现场出现连续变色、关灯再亮或自动切换颜色，请记录为：

```text
UNSTABLE_MULTIPLE_COMMANDS
```

并说明：

```text
控制链路已触达设备，但本轮出现多条控制命令叠加，不能作为稳定验收。下一轮必须只发送一次请求并等待现场确认。
```
