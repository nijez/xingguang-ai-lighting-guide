# 馨光 AI 设计灯光最简安装维护面板

这是独立封测切片，不替换正式内测版，也不继承旧封测维护工具链。

目标只有一个：在腾讯云龙虾应用模板初始化后，通过一个终端菜单完成安装和日常重装维护，不再依赖龙虾对话执行配置命令。

菜单只包含：

1. 一键安装
2. 安装 / 修复基础环境、龙虾与 Miloco 2.0
3. 配置 Miloco 使用的 MiMo API Key
4. 绑定米家账号 / 选择家庭
5. 安装 / 修复馨光 Skill
6. 查看状态
7. 一键关闭 Miloco 自动定时任务（减少 MiMo 用量）
8. 重新开启 Miloco 自动定时任务

明确不包含：

- 分层验收和诊断套件；
- 灯光测试、设备匹配和场景生成；
- 龙虾对话提示词改写；
- 自建定时监控、用量统计、诊断包和实验工具；
- 自动控制任何灯光设备。

## 配置边界

- 腾讯云龙虾应用设置里的模型 Key：属于应用模板初始化，仍在腾讯云页面填写。
- Miloco 使用的 MiMo API Key：在本终端面板中隐藏输入并写入，不在龙虾对话里粘贴。
- 米家授权：在终端生成授权链接，用户完成授权后把授权码粘贴回终端。
- 家庭选择：在终端读取真实家庭列表；只有一个家庭时直接继续，多个家庭时必须由用户选择，不自动使用第一个家庭。
- 馨光 Skill：完成家庭选择后由终端面板安装。

## 为什么把三个配置步骤放进终端

米家授权、家庭选择和 Miloco 使用的 MiMo API Key 都是确定性配置。放在龙虾对话中会受到模型复述、改写或工具调用偏差影响；放进终端面板后，输入、校验和失败结果都由脚本直接控制。

龙虾只保留最终用途：安装完成后接收用户的自然语言灯光需求。本面板不修改龙虾提示词，也不参与灯光效果生成和设备选择。

一键安装的固定顺序：

```text
检查腾讯云龙虾模板环境
→ 检查基础环境并安装 / 修复龙虾与 Miloco 2.0
→ 配置 Miloco 使用的 MiMo API Key
→ 绑定米家账号
→ 选择家庭
→ 安装 / 修复馨光 Skill
```

面板不会修改腾讯云应用设置里的模型 Key，也不会写入旧版龙虾对话规则。

封测运行方式（发布后）：

```bash
curl -fsSL https://nijez.github.io/xingguang-ai-lighting-guide/closed-beta/simple-panel/xinguang -o xinguang && bash xinguang
```

首次运行后会安装本地命令：

```bash
xinguang
```

版本锁定：

- 面板：`2026-07-22.11`
- Miloco 安装后端基线：`2026-06-25.32`（调用 `miloco` 专用动作，包含模板依赖检查、龙虾后台服务检查和 Miloco 安装）
- Miloco 官方安装器：推荐版本与 required 版本均固定为 `2026.7.17`（项目真机兼容清单，不解析官方 release 文案），安装前校验固定 SHA256
- 精简 Skill 安装后端：`2026-07-22.1`
- 馨光 Skill 文件：`4.0.1`，安装前校验固定 SHA256

Miloco 版本策略：

- 未安装：安装推荐版本 `2026.7.17`。
- 已安装版本等于推荐版本，且服务运行中、OpenClaw 插件已注册：跳过安装。
- 已安装版本低于 required 版本：在执行「安装/修复」或「一键安装」时自动升级到推荐版本。
- 已安装版本达到 required 版本但低于推荐版本，且服务与插件正常：只提示可更新，由用户确认后手动更新，不自动升级。
- 已安装版本高于推荐版本：禁止自动降级；服务与插件均正常时跳过，任一不正常则拒绝处理并提示人工核实。
- 官方安装器若明确触发已确认的插件依赖解析异常，面板只对这一种错误启用兼容安装：使用双 SHA256 校验的官方插件包，在面板自有目录安装依赖，再通过 OpenClaw 官方 `--link` 方式注册；其他安装错误仍立即停止。

Miloco 官方定时任务开关（`scheduler.enabled`，v2026.7.17 起提供）：

- 状态页显示当前定时任务为「开启 / 关闭 / 当前版本不支持 / 无法确认」。
- 菜单 7 一键关闭：会关闭 Miloco 官方的感知摘要、家庭巡检、记忆整理、习惯洞察这四项周期任务，不影响用户自建的其他定时任务；执行前需用户确认。
- 菜单 8 重新开启：恢复上述四项官方周期任务。
- 两者都只调用 `miloco-cli config set/get scheduler.enabled`，写入后立即复核，再重启龙虾后台服务使其生效；只用 `openclaw cron list` 做只读确认，不新建、不误删任何定时任务。
