# 馨光 Skill 安装与灯光测试方案

本文是第二阶段内测方案。第一阶段安装流程、安装脚本和页面主流程保持不变。

## 第二阶段目标

让龙虾通过馨光 Skill 控制馨光设备。用户不需要选择产品类型、设备编号或设备型号，只需要先确认米家家庭，然后直接说灯光需求。

第二阶段只保留：

1. 终端一键安装阶段安装基础环境，并预置馨光 Skill 本地安装器。
2. 回到龙虾绑定米家账号。
3. 如果有多个米家家庭，让用户选择馨光设备所在家庭。
4. 用户发送「安装馨光 Skill」。
5. 龙虾执行本地安装器，启动馨光本地灯控服务。
6. 用户直接说灯光需求。
7. 龙虾通过馨光 Skill 执行灯光效果。

## 多家庭选择

多家庭选择是必须保留的安全步骤。

1. 如果只有一个家庭，可以直接继续。
2. 如果有多个家庭，必须展示家庭列表。
3. 用户选择家庭后再继续。
4. 不允许自动使用第一个家庭。

用户可见文案：

```text
如果你的米家账号下有多个家庭，请选择馨光设备所在的家庭。
```

## 用户主流程

完成终端一键安装后，先绑定米家账号。如果米家账号下有多个家庭，先选择馨光设备所在的家庭。之后发送「安装馨光 Skill」。安装成功后，用户可以直接告诉龙虾想要的灯光效果。

示例：

```text
客厅来个马尔代夫的海边日落。
二楼客厅来一个森林晨光。
全屋灯带做一个朋友聚会氛围。

```

## 灯光测试口径

灯光请求发出后，龙虾只提示用户现场观察。

```text
灯光请求已发送，请观察灯光是否发生变化。
如果已变化，请回复：已变化。
如果未变化，请回复：未变化。
```

用户回复“已变化”时：

```text
测试成功。

如果当前效果满意，也可以说：保存当前灯光效果到快照 3。
```

用户回复“未变化”时：

```text
测试未通过，请联系工作人员处理。
```

不要继续自动诊断，不要自动重试，不要自动关灯或切换颜色，不要控制非目标设备。

## 研发修复项

以下问题记录为研发修复项，不进入用户主流程：

1. 本地灯控服务运行路径兼容。
2. 本地授权信息一致性。
3. 返回状态与现场结果不一致。
4. 设备识别边界。
5. 本地灯控服务运行兼容。

## 验收标准

1. 馨光 Skill 已安装。
2. 用户可以选择馨光设备所在米家家庭。
3. 龙虾能读取该家庭下的设备列表。
4. 用户可以用自然语言提出灯光需求。
5. 龙虾能通过馨光 Skill 发出灯光请求。
6. 用户现场观察到灯光变化后，回复“已变化”，龙虾返回“测试成功。”。
