项目目标

最终目标是让用户通过龙虾 / OpenClaw 对话，用自然语言描述灯光场景，例如“马尔代夫凌晨”“森林清晨”“海边日落”。后续馨光 Skill 会提取适合淡彩光的两个颜色值，再通过 Miloco 2.0 控制馨光设备。

适用场景

腾讯云轻量应用服务器创建服务器时选择 OpenClaw 应用模板。
龙虾 / AgentChat 可打开能进入 OpenClaw 对话页，并能把指令发给龙虾执行。
馨光淡彩光控制当前不启用摄像头视觉感知，重点是自然语言灯光控制底座。

服务器配置建议

本项目主要用于馨光 AI 淡彩光控制，不启用摄像头视觉感知能力，因此服务器配置要求低于 Miloco 完整全屋 AI 方案。

用途建议配置说明
测试 / 内测2 核 2G 内存，40G 磁盘可尝试部署。
稳定演示 / 客户试用2 核 4G 内存，60G-80G 磁盘推荐。
长期运行 / 多插件 / 多用户4G 以上内存，80G 以上磁盘根据实际运行反馈升级。

如果只是馨光淡彩光控制，通常不需要高配服务器。如果后续启用摄像头识别、多模态感知、更多 OpenClaw 插件或多人同时使用，再提高配置。

创建腾讯云轻量应用服务器

1

选择应用模板

服务器重装系统或新建实例时选择 OpenClaw 应用模板。本教程默认 OpenClaw 已经预装。

2

打开 OpenClaw / AgentChat / 龙虾

进入腾讯云提供的 OpenClaw 页面，确认能看到 AgentChat 对话框。

3

发出一键部署指令

复制下一节的完整指令发给龙虾。它会下载固定脚本，后台执行，并主动汇报进度。

把一键部署指令发给龙虾

下面这份指令已经清理成单一流程：下载脚本、检查版本、轻量启动部署、观察进度和最终汇报都只保留一份。

默认参数：RUN_SYSTEM_UPGRADE=0、OPENCLAW_UPDATE=0、INSTALL_EXTRA_PLUGINS=0、INSTALL_NONINTERACTIVE=1。OpenClaw 已安装且满足最低兼容版本时跳过升级；低于最低兼容版本时会自动升级。

查看安装进度

如果对话窗口没有实时刷新，安装仍可能在后台运行。发送下面这份指令可以区分 RUNNING、FINISHED 和 EXITED_BUT_INCOMPLETE。
